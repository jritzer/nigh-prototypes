Nigh Icon System

Cross-app icon audit — Enterprise · Pro · Drops (consumer) · Website
Last updated March 31, 2026
Core issue: The Nigh Bug is a filled icon (fill="currentColor", no stroke) while all other Nigh icons are stroke icons (stroke-width 2, fill=none). At equal pixel sizes, filled icons appear visually lighter and more complex than stroke icons. The Nigh Bug also has more intrinsic detail (N letterform, shadow ellipse, dot) than any of the surrounding icons, which makes it feel like a different visual system. Two paths forward are shown below.

Option A — Full Nigh Bug with N, size-compensate Current approach

The full Nigh Bug (with N inside) at a larger size than stroke icons. Enterprise nav now uses 28px Bug vs 20px stroke icons. The N letterform is important to brand identity — Nigh Drops is a new category that deserves the full mark.

Both at 20px
Stroke 20px · Bug 24px

Option B — Outline-only Bug (stroke version) Recommended

Use only the outer shell of the Nigh Bug as a stroke icon — no N inside, no shadow. Matches the weight and style of every other icon. The shape is still instantly recognisable as the Nigh mark.

Nigh Website — Icon & Logo Usage

Website uses the full multi-color Nigh wordmark — not the icon system

The nigh.com website uses inline multi-path SVGs for the Nigh logo (multi-color: indigo #6226E9, blue #0E4EF5, cyan #41EFF9, pink #EC1977, white). This is the full brand lockup, not a UI icon. It does not participate in the stroke icon system used by Enterprise and Drops. Functional UI elements on the website (buttons, nav links) are text-based — no icon tokens are used in the main site.

nigh
The website logo uses 5 overlapping color fills to render the Nigh mark. This is a presentation asset, not a UI icon — it should never be used in nav or icon grids.
