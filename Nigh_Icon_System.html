Nigh Icon System

Cross-app icon audit — Enterprise · Pro · Website
Last updated March 31, 2026
Core issue: The Nigh Bug is a filled icon (fill="currentColor", no stroke) while all other Nigh icons are stroke icons (stroke-width 2, fill=none). At equal pixel sizes, filled icons appear visually lighter and more complex than stroke icons. The Nigh Bug also has more intrinsic detail (N letterform, shadow ellipse, dot) than any of the surrounding icons, which makes it feel like a different visual system. Two paths forward are shown below.

Option A — Keep Bug as-is, size-compensate Current approach

The full Nigh Bug (with N inside) at a larger size than stroke icons. Enterprise nav currently uses 24px Bug vs 20px stroke icons.

Both at 20px
Stroke 20px · Bug 24px

Option B — Outline-only Bug (stroke version) Recommended

Use only the outer shell of the Nigh Bug as a stroke icon — no N inside, no shadow. Matches the weight and style of every other icon. The shape is still instantly recognisable as the Nigh mark.
